= How to read this document

Lines that begin with "=" are headers. Nested headers have more "="
characters. For example:

= Topic to discuss

Some text about the topic.

== Some related topic that warrants its own heading

More text about a related topic of topic.

Lines that begin with "@@@" can be considered notes or additional information
that are not part of the flow of discussion. For example:

@@@
It is worth knowing that this function is a general GCC function that was
removed for AmigaOS4
@@@

Underscores used like _this_ represent emphasis.

Any references to source material look like "(source N)", where N is a number.
"source N" can then literally be searched.

Any references markers to other locations look like "(see [[some_text]])",
where "[[some_text]]" can literally be searched.

Any "(TODO:)"s represent the same thing as they do in code; something that needs
to be finalised in the document.

In general, the way the patches are documented look like this:

someFile.c: (1) this change does this, because this. (TODO: why like that?)

            (2) this other change does this because of the other.

In some cases, it may merely look like a different more natural language
version of the actual patch file, but this document attempts to both declare
the change, in English, and reason about that change; why is it necessary for
AmigaOS4? If there are questions to be answered, then "(TODO:)" markers ask
questions. The actual order of the patch file does not necessarily correlate
to the order as in the example for "someFile.c" above. It may be that (2)'s
diff is applied before (1)'s diff.

= Primer

In order to understand how the compiler and linker work for AmigaOS4 it is
probably a good start to look at the patch files that are applied to the
compiler and linker.

The source repository is `git@github.com:AmigaLabs/adtools.git'. There is a
tool called `GILD' that basically pulls down GCC and BINUTILS from the
official (GNU/mirrors) repositories and then automatically applies Amiga
specific (though, the patches also contain MorphOS etc related changes)
changes.

The easiest thing to do is to look at the patches that are applied to the
earliest versions of the components, since the earlier, the simpler. This
document looks at GCC 4.9 and BINUTILS 2.23.2.

It is worth noting that the order of the patches are purely alphabetical. It
may be more clear to look at certain ones before others so that a coherent
story can be made. The author has attempted to do this.

= Compiler

The majority of the changes are built from
`0001-Changes-for-AmigaOS-version-of-gcc.patch'. In this patch, official files
are changed and new ones are added. The newly added files added to the
gcc/config directory and are the target Amiga specific files such as the
linker script.
